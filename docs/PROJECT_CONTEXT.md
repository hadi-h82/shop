# زمینهٔ پروژه

## معرفی

این پروژه، فرانت‌اند یک فروشگاه فارسی و راست‌به‌چپ (RTL) برای فروش لوازم قنادی و شیرینی‌پزی است. برنامهٔ Angular در پوشهٔ `shop-client/` قرار دارد.

## هدف

ساخت فروشگاهی ساده، تمیز، حرفه‌ای و قابل‌توسعه برای نمایش محصولات، انتخاب کالا و طی‌کردن جریان اولیهٔ خرید.

## مخاطبان اصلی

بر اساس محتوای فعلی کد، مخاطبان فروشگاه شامل قنادان، شیرینی‌پزها، فعالان تهیه و تزئین کیک و شیرینی، و علاقه‌مندان به ابزار و لوازم شیرینی‌پزی هستند.

## دامنه و استقرار

- دامنهٔ اصلی موردنظر: `noviraone.ir`
- استقرار فعلی فرانت‌اند: Runflare
- **نیازمند تصمیم / هم‌راستاسازی:** در سرویس‌های SEO و صفحات فعلی، canonical URLها هنوز با `https://sevart.ir` ساخته می‌شوند. پیش از انتشار نهایی باید دامنهٔ canonical، متاتگ‌ها و تنظیمات استقرار با دامنهٔ اصلی هم‌راستا شوند.

## فناوری‌های اصلی

- Angular `22.0.x` با Angular CLI/Build `22.0.5`
- Angular SSR `22.0.5` و Express `5.1.0`
- Angular Material و CDK `22.0.4`
- TypeScript `~6.0.2` و RxJS `~7.8.0`
- SCSS، Tailwind CSS `4.3.2` و فونت Vazirmatn `33.0.3`
- npm `11.16.0`
- Vitest `4.0.8` برای تست واحد

معماری فعلی از کامپوننت‌های standalone، `inject()`، `signal()` و `computed()` استفاده می‌کند. رندر سمت‌سرور با `@angular/ssr` و فایل‌های `main.server.ts`، `server.ts`، `app.config.server.ts` و `app.routes.server.ts` پیکربندی شده است.

## صفحات و قابلیت‌های موجود

- صفحهٔ خانه با اسلایدر خودکار، دسته‌بندی‌های فعال و نمایش محصولات منتخب
- فهرست همهٔ محصولات
- فهرست محصولات هر دسته‌بندی از مسیر `categories/:slug`، همراه با محتوای راهنمای خرید برای دسته‌ها
- جزئیات محصول از مسیر `products/:id`
- کارت محصول و افزودن محصول قابل‌فروش به سبد خرید
- سبد خرید با افزایش/کاهش تعداد، حذف کالا، محاسبهٔ تعداد و مبلغ کل، و پیوند به تسویه‌حساب
- صفحهٔ تسویه‌حساب با فرم نام، موبایل، استان، شهر، کد پستی و نشانی و اعتبارسنجی اولیه
- صفحات lazy-loaded ورود، ثبت‌نام، فراموشی رمز عبور و بازنشانی رمز عبور با فرم‌های Reactive Forms و اعتبارسنجی سمت کاربر
- سوییچ Light/Dark Theme و ذخیرهٔ انتخاب تم در مرورگر
- به‌روزرسانی title، description، Open Graph و canonical URL در صفحات فروشگاه

## وضعیت داده‌های Mock

- دادهٔ محصولات و دسته‌بندی‌ها در `src/app/core/mock-data/` و با نام‌های `MOCK_PRODUCTS` و `MOCK_CATEGORIES` نگهداری می‌شوند.
- صفحات خانه، محصولات، دسته‌بندی و جزئیات محصول مستقیماً از همین داده‌ها استفاده می‌کنند.
- مسیرهای prerender سمت‌سرور نیز از همین Mockها برای ساخت مسیر محصولات و دسته‌بندی‌ها استفاده می‌کنند.
- سبد خرید فقط در `localStorage` مرورگر نگهداری می‌شود و داده‌ای به سرور ارسال نمی‌شود.
- ارسال فرم تسویه‌حساب و فرم‌های احراز هویت هنوز API فراخوانی نمی‌کند و در وضعیت فعلی دادهٔ معتبر را فقط در `console.log` ثبت می‌کند.

## مسیر آیندهٔ Backend

برنامهٔ اعلام‌شده، اتصال فرانت‌اند به Backend مبتنی بر **ASP.NET Core Web API** و **SQL Server** است. این اتصال در کد فعلی پیاده‌سازی نشده است.

موارد موردانتظار برای این اتصال شامل جایگزینی داده‌های Mock با API، مدیریت محصول و دسته‌بندی، احراز هویت، ثبت سفارش و نگهداری داده‌ها در SQL Server است.

- **نیازمند تصمیم:** قرارداد API، مدل‌های درخواست/پاسخ، روش احراز هویت و مدیریت نشست، درگاه پرداخت، مدل سفارش و موجودی، سیاست خطا و pagination هنوز در کد فعلی مشخص نشده‌اند.
- **نیازمند تصمیم:** تعیین محیط‌ها، URLهای API، CORS، روش نگهداری secretها و فرایند استقرار Backend.

## الزامات اصلی

- **SSR:** هر استفاده از `window`، `localStorage` و دیگر APIهای صرفاً مرورگر باید با `isPlatformBrowser` و `PLATFORM_ID` محافظت شود. برای تعامل با DOM از `DOCUMENT` استفاده شود.
- **Responsive و Mobile:** همهٔ اجزای رابط باید در اندازه‌های موبایل و دسکتاپ قابل‌استفاده باشند.
- **RTL و فارسی:** جهت صفحه `rtl` و زبان صفحه `fa` است؛ CSS و چیدمان باید این قرارداد را حفظ کنند.
- **Light Theme و Dark Theme:** هر تغییر بصری باید هر دو تم را پشتیبانی کند و انتخاب تم کاربر را حفظ کند.

## موارد نیازمند تصمیم

- دامنهٔ نهایی canonical و جایگزینی `sevart.ir` موجود در کد با `noviraone.ir`.
- جزئیات محصول، قیمت‌گذاری و گزینه‌های محصول؛ داده‌های Mock موجود عمدتاً قیمت `0` و گزینه‌های خالی دارند.
- اتصال واقعی احراز هویت، ثبت سفارش، پرداخت، ارسال و مدیریت حساب کاربری.
- قرارداد و طراحی کامل Backend مبتنی بر ASP.NET Core Web API و SQL Server.
